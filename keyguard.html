KeyGuard

Passwords protect the moment you log in. KeyGuard protects everything after.

★ Best Automation — Catapult 2026

Overview

Built solo in 36 hours, KeyGuard is a continuous authentication system that monitors the typing rhythm of whoever is at the keyboard — not just at login, but throughout the session. It captures 26 behavioral features per active typing burst and scores every 40-keystroke window against the enrolled user's baseline. Two consecutive anomalous windows triggers a session lock. All processing is local; no keystrokes, no biometric data, nothing leaves the machine.

Detection Pipeline

Keystrokes are segmented into active bursts by pauses above a threshold. Each burst contributes to a rolling 40-keystroke window. From each window, 26 features are extracted: mean and standard deviation of dwell time and inter-key flight time, pause rate, backspace rate, burst WPM, and digraph timing for the 20 most common English letter pairs (th, he, in, er, an, …). The window is then scored by three independent detectors.

Ensemble Model

Isolation Forest (200 trees) serves as the primary detector, trained on the enrolled user's baseline and scoring each new window for isolation depth. One-Class SVM with RBF kernel provides a corroborating signal. Z-score anomaly detection on a selected feature mask acts as the decision-maker, catching cases where the other two models agree but the most discriminating features are still anomalous.

Two consecutive anomalous windows triggers a lock. Single anomalous windows are logged but do not lock — this eliminates false positives from momentary distraction or atypical typing bursts.

Online Adaptation

The enrolled baseline updates via exponential moving average (α = 0.005) after each verified window. Only windows that pass the anomaly threshold update the model — an intruder cannot slowly shift the baseline to accept their typing pattern by staying just below the detection threshold.
